# AI 에이전트 고도화 검토

## 현재 상태 (2026-04-22 기준)

현재 에이전트는 LangGraph 기반이지만 실질적 볼륨이 작음.

| 에이전트 | 실제 LLM이 하는 것 | 스텝 수 |
|---|---|---|
| course_list_agent | 15개 후보 중 3개 선택 | 1~2 스텝 |
| course_detail_agent | 여행 내러티브 생성 | 1 스텝 |

Python이 사전 선별(50→15)과 설화 매핑을 모두 처리하기 때문에 LLM의 자율적 행동 범위가 좁음.

---

## 검토 중인 업그레이드 방향

### 방향 A — 대화형 코스 조정 에이전트
코스 3개를 보여준 뒤 사용자가 채팅으로 피드백하면 에이전트가 실시간 수정.

```
사용자: "신화 이야기는 좋은데 해안가 장소가 더 있으면 좋겠어"
에이전트: 현재 코스 분석 → 해안 설화 검색 → 장소 교체 후보 탐색 → 수정안 제시
```

- 멀티턴 대화 루프
- 도구 3~4개 (설화 검색, 장소 교체, 코스 검증)
- iOS에 채팅 UI 추가 필요

### 방향 B — 멀티 에이전트 파이프라인
```
[분석 에이전트] → 취향 해석
[코스 설계 에이전트] → 장소 순서·거리 최적화
[설화 큐레이션 에이전트] → 설화 연결 스토리 구성
[검토 에이전트] → 품질 평가 후 재조정 요청
```

- 에이전트 간 오케스트레이션
- 논문·발표 구조도 풍부해짐

## 결정 보류 이유

현재 우선순위는 서비스 완성도(앱 기능 추가, 안정화).
에이전트 고도화는 서비스 개발 완료 후 재검토.
