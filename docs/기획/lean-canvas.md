# 탐라담 (가칭) — Lean Canvas v0.1

> 작성일: 2026-05-21
> 1인 개발자: 조익준 (경희대 컴공)
> 현재 단계: 졸업 프로젝트 MVP / 발표 임박
>
> **이 문서의 목적**: 산발적 기능 추가를 막고, 매 작업 시 "이게 우리 핵심 가치와 맞는가?" 판단 기준을 만드는 것.
> `❓` 표시는 사용자(조익준)가 결정해줘야 하는 항목.

## 0. One-Liner

**"제주에 잠든 설화를, 위치 기반 동화 여행 코스로 되살리는 AI 가이드 앱"**

(❓ 어색하면 톤·단어 다듬기. 핵심 키워드: *제주 / 설화 / 위치 기반 / AI / 동화체 가공*)

---

## 1. Problem (top 3)

### P1. "제주 여행 가도 정형화된 명소만 돌고 끝"
- 카페·해변·오름 위주의 인스타용 코스. 깊이가 없음
- 같은 장소를 다녀와도 "왜 거기 갔지?" 가 잘 안 남음

### P2. "제주의 설화·전설·민속신앙은 풍부한데, 일반 여행자는 접근 불가"
- 설화 데이터는 학술적 어투("~의 내력을 담은 무속신화이다") 그대로
- 한 번 읽고 5초 만에 닫음. 호기심 자극 X
- *어떤 장소에서 어떤 이야기가 있는지* 매칭 정보가 흩어져 있음

### P3. "맞춤형 코스를 짜기 어렵다"
- 가족/연인/혼자 등 동행 구성에 따라 맞는 코스가 다른데 검색·필터로 못 풂
- 블로그·유튜브 베끼는 게 한계

**현재 대안 (불만족):** 네이버 블로그, 트립어드바이저, 카카오맵 즐겨찾기. 모두 *큐레이션 없고 / 설화 없고 / 동선 최적화 약함*.

---

## 2. Solution (top 3)

### S1. **LLM 동화체 리라이팅**
- 학술 어투 설화 → "옛날 옛적, 하늘나라에 옥황상제의 막내딸이…" 같은 5-7페이지 동화 스토리북
- 인스타 스토리 UX 풀스크린 뷰어
- 장소-설화 한 줄 연결("이 마을의 본향당 신을 모시는 이야기")

### S2. **AI 코스 추천 (LangGraph 에이전트)**
- 사용자 취향 진단 → 카테고리 점수 → Top 3 코스 자동 생성
- 같은 취향이어도 매번 다른 조합 (랜덤 풀)

### S3. **위치 기반 탐험 모드 + 동행자 캐릭터**
- 코스 진행 중 도착 자동 감지 → 그 장소 설화 푸시
- 동행자 캐릭터 (LLM 챗) 가 함께 여행하는 느낌
- 완료 시 LLM이 여행 일지 + 민화 이미지 생성

---

## 3. Unique Value Proposition (UVP)

> **"학술 데이터를 동화로 바꿔주는 AI 가이드. 제주 갈 때 친구가 아니라 *심방(제주 무속의 사제)* 과 함께 가는 느낌."**

**왜 다른가:**
- 일반 여행앱은 "어디 갈까?"만, 우리는 "왜 거기 가야 하는가?" 답함
- 한국에 제주 설화 전체를 LLM으로 가공해 일반 사용자에게 노출하는 제품은 사실상 없음
- 위치 기반 자동 트리거 + 동행자 캐릭터 조합은 *살아있는 가이드* 경험

(❓ 위 표현 마음에 안 들면 알려줘. "심방" 같은 토속어 강하게 가는 게 좋은지, 더 부드럽게 갈지)

---

## 4. Unfair Advantage

### 단기 (1년)
- **제주 설화 데이터셋 + 좌표 매핑** (KTO TourAPI + 본풀이/민담 약 250+개)
- **LLM 가공 캐시 자산** — hook/connection/스토리 페이지 영구 캐시 → 신규 진입자가 똑같이 만들려면 비용·시간 큼
- **개발자 본인이 제주 출신** — 로컬 컨텍스트·언어·정서 이해

### 중장기
- ❓ **지자체/문화재단 제휴 가능성?** (제주문화원, 제주관광공사 등 — 데이터 협력)
- ❓ **사용자 리뷰/방문 기록 누적 데이터** — 어떤 설화·장소가 인기 있는지 학습

(❓ 진짜 방어 가능한 게 무엇인지 — 지금은 약함. 사용자 답: …)

## 7. Revenue Streams

❓ **이게 가장 큰 결정 사항. 졸업 후 어디로 가고 싶은지에 따라 달라짐.**

### 옵션 A: 완전 무료 + 광고
- 데이터 노출량 위주
- 광고 단가 낮음 (여행앱은 1000회 노출당 $0.5~2)

### 옵션 B: 프리미엄 (Freemium)
- 무료: 홈/지도/일부 설화 + 광고
- 유료(월 ₩2,900 또는 일회성 ₩4,900): 모든 설화 스토리북 + 코스 무제한 + 광고 제거 + 동행자 캐릭터 풀버전

### 옵션 C: B2B / 지자체 라이선스
- 앱 자체는 무료, 지자체·관광공사가 콘텐츠 비용 지불
- 가장 큰 매출 가능, 영업 부담 큼

### 옵션 D: 일회성 코스 판매
- 큐레이션된 프리미엄 코스 ₩1,900~₩4,900
- (예: "할망이 들려주는 제주 무속 1박2일")

(메인 추천: 졸업 후 출시 의지 있으면 **B + D 하이브리드**, 시연만 목적이면 **A**. 사용자 답: …)

## 8. Cost Structure

### 고정비 (월)
- ❓ Firebase (Auth + Firestore + Storage): 사용자 1만 명까지 거의 0원, 그 이후 ~$30/월
- ❓ FastAPI 서버 (Render/Fly.io): $7~25/월
- KTO TourAPI: 무료 (할당량 내)
- 도메인·App Store 개발자 계정: 연 $99 (Apple) + $20 (도메인)

### 변동비
- **OpenAI API (LLM 호출)**: 핵심 비용. 설화 hook/connection/story 영구 캐시라 신규 사용자 = 신규 비용 거의 없음. 새 코스/리뷰 등 동적 LLM만 비용 발생
- **이미지 생성** (일지 민화): nano-banana 또는 DALL-E. 한 번 생성 = ~$0.04. 사용자 1명당 1~3장 → $0.04~0.12

### 핵심 비용 동인
- LLM 토큰 (특히 코스 추천 에이전트 — 매 호출마다 LLM)
- 이미지 생성 비용

### CAC
- ❓ 추정 불가. SNS 유기 노출 위주면 0에 가까움, 광고 집행 시 한국 모바일 앱 평균 CAC ₩2,000~5,000

## 9. Key Metrics

### Activation (이 앱이 "쓸 만하다"고 느끼는 순간)
- **"첫 설화 스토리북 끝까지 본 사용자 비율"** — 핵심 가치 체험 지표
- 목표: 신규 사용자의 50%+

### Retention
- **D7 retention** (7일 후 재방문) — 일반 여행앱 평균 10~15%
- **"코스 1개 이상 저장한 사용자"** — 의미 있는 액션

### Revenue (옵션 B/D 갈 경우)
- ARPU (Average Revenue Per User)
- 유료 전환율

### North Star Metric
- ❓ 후보 1: **"AI 코스 시작 → 완료까지 갔고 일지가 생성된 횟수"** (full-funnel completion)
- ❓ 후보 2: **"풀스크린 스토리 뷰어를 1회 이상 읽은 MAU"** (콘텐츠 핵심 가치 체험)

(메인 추천: **후보 1** — 핵심 동선의 전체 완주가 곧 가치 증명)

---

## 핵심 가설 (검증 필요)

1. **사용자가 학술 어투 설화 대신 동화체를 선호한다** — 시연/사용성 테스트로 검증
2. **위치 기반 자동 트리거 + 동행자 캐릭터는 단순 가이드북보다 몰입감 높다** — 미검증
3. **유료 전환 의향 사용자가 일정 비율 존재한다** — 옵션 B/D 갈 경우 출시 후 검증
4. **제주 설화는 "한 번 보고 끝"이 아니라 다회 방문 유발한다** — D7 retention 으로 검증

## 한 번에 고민할 의사결정 5개 (`❓` 요약)

1. **타겟 (Customer Segment)**: 20~30대 자유여행자 / 가족 / 외국인 중 1순위는?
2. **수익 모델**: A 완전무료 / B 프리미엄 / C B2B / D 코스 판매 — 어느 길?
3. **시연/실서비스 목표**: 졸업 시연만 OK인가, 출시도 진심인가?
4. **방어 자산**: 제주 설화 데이터셋 외에 진짜 "다른 사람이 못 따라할 것" 이 무엇인가?
5. **초기 100명 모을 채널**: SNS / 블로그 / 지자체 / 그 외?

이 5개만 답해주면 lean canvas v1.0 으로 확정하고, 향후 모든 기능 결정 시 "이게 이 캔버스의 어느 칸에 기여하는가?" 기준으로 판단합니다.
