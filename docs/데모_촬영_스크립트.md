# 데모 촬영 스크립트 — 졸업 평가 제출용

> **목표**: 시뮬레이터에서 3~5분 분량으로 시연 10단계 + 홈 화면 + 민화 일지까지 끊김 없이 보여준다.
> **사용법**: 촬영 직전에 이 문서를 옆에 띄워두고 씬 순서대로 따라간다.

---

## 0. 사전 준비 체크리스트 (촬영 시작 5분 전)

### 환경
- [ ] 백엔드 실행 — 터미널에서 `cd ~/Desktop/keepdev/졸프 && ./start_dev.sh`
- [ ] 헬스 확인 — 브라우저 `http://localhost:8000/docs` 또는 `curl http://localhost:8000/health` → 200 OK
- [ ] `.env` 키 정상 (OPENAI_API_KEY, TOUR_API_KEY) — 백엔드 로그에 "Loaded" 확인
- [ ] Xcode 시뮬레이터: **iPhone 15 Pro** (또는 16 Pro) 선택, iOS 17 이상
- [ ] 시뮬레이터 위치 시뮬레이션: **Features → Location → Custom Location** 또는 **City Run** 켜둠
- [ ] 시뮬레이터 **음소거 OFF** (페르소나 TTS가 영상에 들려야 함) — Mac 시스템 볼륨도 50% 이상
- [ ] 앱 데이터 깔끔하게 — 필요 시 시뮬레이터 **Erase All Content and Settings** 후 재빌드

### 녹화
- [ ] QuickTime Player → **새로운 화면 기록** → 시뮬레이터 창 영역 선택
- [ ] 또는 macOS 단축키 `Cmd + Shift + 5` → 선택 영역 기록
- [ ] **마이크 입력 ON** (나레이션 추가하려면) 또는 OFF (자막만 쓸 거면)
- [ ] 디스크 여유 공간 5GB 이상

### 멘탈 리허설
- [ ] 한 번 빠르게 처음~끝까지 클릭 동선 시뮬레이션 (녹화 안 한 상태로)
- [ ] 어느 시점에 GPS 도착 버튼 누를지 정확히 기억
- [ ] 일지 생성 25~30초 로딩 동안 무슨 말 할지 미리 정해놓기

## 1. 권장 영상 길이

| 구간 | 시간 |
|---|---|
| 인트로 + 홈 화면 | 20초 |
| 코스 만들기 (씬 2~3) | 40초 |
| 미리보기 + 탐험 시작 (씬 4~5) | 30초 |
| 첫 장소 도착 + 동행자 채팅 (씬 6~9) | 80초 |
| 두 번째 장소 (짧게) | 20초 |
| 탐험 마치기 + 일지 생성 (씬 11~12) | 50초 |
| 공유 + 내 코스 하이라이트 (씬 13~14) | 20초 |
| **총합** | **약 4분 (240초)** |

## 2. 씬 단위 촬영 스크립트

각 씬 형식:
- **무엇을 보여주나** — 화면 + 조작
- **나레이션/자막** — 한 줄
- **예상 시간**

---

### 씬 1. 앱 첫 진입 — 홈 화면

- **보여주기**: 앱 아이콘 탭 → 스플래시 → 홈 탭 자동 선택. 화면 위쪽 **"오늘의 설화" 카드** + 아래쪽 **제주 지도 + 설화 핀**. 지도 살짝 핀치/스크롤로 핀 분포 보여줌.
- **자막**: "제주 설화 기반 AI 여행 가이드, 탐라담."
- **시간**: 15초

### 씬 2. 코스 만들기 진입 — 카테고리/지역/일수 선택

- **보여주기**: 하단 **"코스 만들기" 탭** 탭. TasteDiscoveryView 진입. 5개 카테고리 중 **메인 1개 + 사이드 1개 직접 선택** (예: 메인 "신화·전설", 사이드 "역사 인물"). 지역 토글, 일수 = 2일 선택 → **"코스 추천 받기"** 버튼.
- **자막**: "좋아하는 설화 카테고리와 여행 일수만 고르면 됩니다."
- **시간**: 25초

### 씬 3. AI 코스 추천 3개 수신

- **보여주기**: 로딩 화면 (ProgressView + "AI가 코스를 추천하고 있어요..." 문구) **5~10초**. 이어서 **Top 3 코스 카드** 표시 → 1순위 카드 탭 → 미리보기 진입.
- **자막**: "10초 안에 취향에 맞는 코스 3개를 받습니다."
- **시간**: 20초

### 씬 4. 코스 미리보기 — 지도 + Day별 장소 + 페르소나

- **보여주기**: CoursePreviewView. 지도 위에 경로 표시, Day 1 / Day 2 탭으로 일정 분리. 각 장소 카드 옆에 **자동 배정된 페르소나** ("마을 할망", "심방" 등) 표시. 시간 여유 있으면 **"바꾸기"** 1회 시연 후 원복.
- **자막**: "코스마다 다른 페르소나의 동행자가 자동 배정됩니다."
- **시간**: 20초

### 씬 5. 탐험 시작 — 지도 이동 경로

- **보여주기**: **"탐험 시작"** 버튼 탭 → ExploreView 진입. 지도 위에 다음 장소까지 **점선 경로** 표시, 하단 다음 장소 카드.
- **자막**: "탐험을 시작하면 지도가 길을 안내합니다."
- **시간**: 10초

### 씬 6. GPS 도착 시뮬레이션

- **보여주기**: 시뮬레이터 우상단 **디버그 메뉴** 또는 화면 내 **"다음 장소 도착" 시뮬 버튼** 탭. 도착 알림 효과 → AI 동행자 오버레이 등장.
- **자막**: "GPS로 도착을 감지하면..."
- **시간**: 10초

### 씬 7. AI 동행자 등장 → 채팅 + TTS

- **보여주기**: 오버레이 탭 1회 → CompanionChatView 진입. **첫 인사가 자동으로 도착** (페르소나 말투, TTS 음성 재생). 사용자 메시지 1개 전송 ("이 마을 이야기 들려줘") → AI 응답 1턴 (TTS 들리도록). 채팅 2~3턴 진행.
- **자막**: "마을 할망이 먼저 말을 걸어옵니다." *(페르소나에 맞게 수정)*
- **주의**: 음성이 영상에 녹음되는지 확인. 안 들리면 시뮬레이터 볼륨 키우고 재녹화.
- **시간**: 45초

### 씬 8. 장소 상세 — 풀스크린 설화 스토리 뷰어

- **보여주기**: 채팅 상단/하단의 **설화 카드 탭** 또는 핀 상세 진입 → **풀스크린 스토리 뷰어**. 좌우 스와이프 5~7페이지, 페이지 인디케이터 확인.
- **자막**: "설화는 인스타 스토리처럼 동화체로 풀어냅니다."
- **시간**: 20초

### 씬 9. 채팅 종료 → 리뷰 바텀시트

- **보여주기**: 스토리 뷰어 닫기 → 채팅으로 돌아옴 → **"대화 마치기"** 또는 자동 종료 트리거. **PlaceReviewSheet** 등장 → 태그 1~2개 ("조용함", "전망 좋음") 선택 → **"남기기"** 탭.
- **자막**: "장소 인상은 태그로 빠르게 남깁니다."
- **시간**: 15초

### 씬 10. 두 번째 장소 도착 (짧게)

- **보여주기**: 지도 화면 복귀 → 다시 **"다음 장소 도착"** 버튼 → 동행자 등장 → 채팅 1턴만 빠르게 → 닫기. (반복임을 강조하되 풀로 다 보여주지 않음)
- **자막**: "두 번째 장소도 동일한 흐름으로."
- **시간**: 20초

### 씬 11. 탐험 마치기 → Day 요약

- **보여주기**: 상단 **"탐험 마치기"** 버튼 → DaySummaryView. **방문 수 배지 + 설화 수 배지** + 동행자 정보 표시.
- **자막**: "여행이 끝나면 그날의 요약이 나옵니다."
- **시간**: 15초

### 씬 12. 일지 생성 → 민화 + 텍스트 일지

- **보여주기**: **"여행 일지 만들기"** 버튼 → 로딩 (25~30초, ProgressView). 로딩 동안 자막으로 "AI가 대화를 일지로 정리하고 민화풍 이미지를 그립니다." → 완성된 **민화 이미지 + 텍스트 일지** 출력.
- **자막**: "대화가 자동으로 민화풍 일지가 됩니다."
- **로딩 시간 활용**: 자막을 두 줄로 나눠 첫 줄은 로딩 중, 두 줄은 완성 시점에 표시.
- **시간**: 40초

### 씬 13. 공유 시트 — ShareLink 노출

- **보여주기**: 일지 화면 상단 **공유 버튼** 탭 → **UIActivityViewController** 시트 등장 (카톡/메모/Twitter 아이콘 보임) → **취소 또는 닫기**로 빠르게 빠져나옴.
- **자막**: "한 번에 SNS로 공유."
- **시간**: 8초

### 씬 14. "내 코스" 탭 자동 이동 + 하이라이트

- **보여주기**: 일지 화면 **"완료"** 탭 → **자동으로 "내 코스" 탭으로 이동** → 방금 끝낸 코스가 **1.5초간 배경 강조** → 강조 풀리며 정상 목록 상태. 마지막 1초 정도 정지.
- **자막**: "방금 끝낸 여행은 자동으로 내 코스에 저장됩니다."
- **시간**: 12초

## 3. 시연 시 절대 보이면 안 되는 것

> **COMPANY.md 시연 실패 조건 기반**

| 금지 | 대응 |
|---|---|
| 500 에러 메시지 | 발생 즉시 녹화 STOP. 백엔드 로그 확인 후 재시작 → 처음부터 재촬영 |
| 빈 화면 10초 이상 | 로딩 UX(ProgressView)가 깔리는지 사전 확인. 깔린다면 OK |
| 동행자가 아무 말 안 함 (무음) | 시뮬레이터 음소거 사전 점검 필수. 무음 시 재촬영 |
| 일지 생성 실패 (에러 또는 빈 화면) | OPENAI 키 만료/한도 사전 확인. 실패 시 백엔드 로그 보고 재시도 |
| 디버그 탭/디버그 버튼이 의도와 다르게 노출 | "다음 장소 도착" 디버그 버튼은 OK (시연용), 그 외 디버그 메뉴는 가리기 |
| 한글 인코딩 깨짐 | 빌드 후 한글 한 번 확인 |
| Firebase 미설정 에러 로그 | 시연 시 GoogleService-Info.plist 없으면 로그인 안 쓰는 흐름으로 시연 (정상 동작) |

## 4. 트러블슈팅 — 촬영 중 사고 시 1줄 대응

| 사고 | 1줄 대응 |
|---|---|
| 동행자 첫 인사가 안 옴 | 채팅 입력창에 아무 말이나 한 번 보내면 응답 옴. 자연스럽게 "한번 말 걸어볼게요" 멘트로 커버 |
| TTS 음성 안 들림 | Mac 시스템 볼륨 + 시뮬레이터 볼륨 동시 키움. 그래도 안 되면 자막으로 대체 ("AI 동행자 음성: ...") |
| 일지 생성 25초 이상 (40초+) 로딩 | 자막에 "AI가 민화풍 일지를 생성하는 중..." 유지. 60초 넘으면 백엔드 로그 확인 후 재촬영 |
| 코스 추천이 빈 결과 (0건) | 카테고리/지역 조합 바꿈 (메인 "신화·전설" + 지역 전체 토글 ON이 가장 안전) |
| GPS 도착 버튼 안 보임 | ExploreView 디버그 모드 빌드인지 확인. Release 빌드면 시뮬레이터 Features→Location→Custom 으로 좌표 직접 입력 |
| Firebase 로그인 시트 의도치 않게 등장 | 시연용 빌드는 로그인 비로그인 둘 다 동작. "건너뛰기" 또는 닫기로 빠져나옴 |
| 백엔드 500 에러 | 터미널에서 `./start_dev.sh` 다시 실행. 다시 처음부터 (씬 1 부터) 촬영 |
| 화면 회전 / 키보드 갑자기 등장 | 시뮬레이터 키보드 토글(`Cmd+K`)로 끔. 회전은 `Cmd+←/→` 으로 복구 |

---

## 5. 촬영 후 체크

- [ ] 영상 길이 3~5분 사이
- [ ] 동행자 음성이 1회 이상 들림
- [ ] 일지 민화 이미지 + 텍스트 둘 다 화면에 잡힘
- [ ] 빈 화면 10초 이상 구간 없음
- [ ] 에러 메시지·디버그 로그 노출 없음
- [ ] 시작과 끝이 자연스러움 (홈으로 시작 → 내 코스 하이라이트로 끝)

---

## 6. 빠른 참조 — 씬 순서 한 줄 요약

```
1. 홈 (지도+핀) → 2. 카테고리 선택 → 3. AI 추천 3개 → 4. 미리보기(페르소나)
→ 5. 탐험 시작 → 6. GPS 도착 → 7. 동행자 채팅(TTS) → 8. 풀스크린 스토리
→ 9. 리뷰 태그 → 10. 두번째 도착(짧게) → 11. 탐험 마치기/요약
→ 12. 일지 생성(민화) → 13. 공유 시트 → 14. 내 코스 하이라이트
```

**총 14씬 / 약 4분 / 졸업 평가 제출용 데모.**
