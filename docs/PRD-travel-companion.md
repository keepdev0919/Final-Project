## 1. Summary

제주 설화 여행 앱의 두 번째 단계로, 코스 계획 이후 실제 여행이 시작되는 순간부터 종료까지의 경험을 설계한다. AI 설화 동행자가 GPS 도착 감지를 기반으로 사용자와 실시간 대화를 나누며 장소에 깃든 설화를 풀어내고, 여행이 끝나면 그 대화를 바탕으로 개인화된 여행 일지를 자동 생성한다. 같은 코스를 여행하더라도 동행자와 나눈 대화에 따라 완전히 다른 여행 기록이 남는 것이 핵심 차별점이다.

## 3. Background

### 현재 상황

지금까지 구현한 기능은 모두 **여행 계획 단계**에 집중되어 있다.

- 지역 선택 (제주도 지도 직접 탭)
- 설화 취향 선택 (Q1: 서사 / Q2: 공간)
- AI 코스 추천 + 설화 매핑
- 코스 미리보기 (Day별 경로, 내러티브, 장소 상세)

코스를 확정하고 나면 앱이 더 이상 할 일이 없다. 사용자가 실제로 제주를 걷는 동안, 폰을 열면 아무것도 없다.

### 핵심 질문

> **"여행 중에 폰을 열면 무엇이 보여야 하는가?"**

### 왜 지금인가?

- **기술적 가능성:** CoreLocation CLCircularRegion을 이용한 지오펜스 도착 감지는 iOS에서 이미 안정화된 기술이다. 백그라운드 위치 권한도 Info.plist에 이미 선언되어 있다.
- **차별화 필요성:** 코스 추천 기능만으로는 기존 여행 앱(트리플, 네이버 지도)과 구분되지 않는다. 여행 중 설화 체험이 우리 앱만의 핵심 가치다.
- **LLM 비용 하락:** GPT-4o를 이용한 실시간 캐릭터 대화와 일지 생성이 현실적인 비용으로 가능해졌다.

### 레퍼런스 분석 요약

| 앱 | 도착 감지 방식 | 트리거 | 여행 후 기록 |
|---|---|---|---|
| 트리플 | 없음 (수동 체크인) | 없음 | 수동 편집 |
| Detour | GPS + iBeacon 지오펜스 | 오디오 자동 재생 | 없음 |
| 포켓몬GO | 지오펜스 40~80m | 시각 변화 → 탭 1회 → 풀스크린 | 없음 |
| 폴라스텝 | WiFi/GPS 체류 감지 | Suggestion 팝업 | 자동 + 수동 혼합 |
| 구글 타임라인 | ML 체류 감지 (완전 자동) | 없음 (사후 알림) | 완전 자동 |

**채택 방향:** Detour(지오펜스 도착 감지) + 포켓몬GO(즉각적 화면 전환) + 폴라스텝(명시적 여행 선언 + 자동 기록) 결합.

**Detour 핵심 교훈:** "기술이 배경으로 물러나야 한다." 채팅을 강제로 띄우는 것이 아니라 탭 1회의 선택권을 주는 것이 인게이지먼트를 높인다.

## 4. Objective

### 목표

여행자가 제주의 설화 장소에 도착하는 순간, 앱이 먼저 말을 걸어오는 경험을 만든다. AI 동행자와의 대화가 여행 기록이 되고, 여행이 끝나면 그 기록이 자동으로 나만의 여행 일지로 완성된다.

### 사용자에게 주는 가치

- **여행 중:** 장소에 도착하면 앱이 알아서 그 장소의 설화 이야기를 꺼낸다. 여행자가 직접 찾아볼 필요 없음.
- **여행 후:** 대화 내용이 자동으로 일지가 된다. 같은 코스를 여행한 두 사람의 일지가 완전히 다른 이야기로 나오는 것이 우리 앱만의 차별점.

### 성공 지표 (Key Results)

| 지표 | 목표 | 측정 방법 |
|---|---|---|
| 여행 시작 전환율 | 코스 확정 사용자의 60% 이상이 "오늘 탐험 시작" 탭 | 로컬 이벤트 로그 |
| 장소 도착 → 채팅 진입률 | 도착 감지 후 탭 1회 전환 70% 이상 | 로컬 이벤트 로그 |
| 장소당 평균 대화 턴 수 | 3턴 이상 | 채팅 기록 분석 |
| 여행 일지 생성 완료율 | 여행 종료 사용자의 80% 이상 일지 생성 | 백엔드 API 호출 수 |

> **참고:** 현재 서비스는 졸업 프로젝트 단계로, 사용자 규모보다 기능 완성도와 경험 품질이 핵심 지표다.

## 5. Market Segment

### 주요 대상

**"제주를 여행하면서 단순 관광 이상의 깊이 있는 경험을 원하는 사람"**

구체적으로:
- 제주 민속·문화에 관심 있는 20~40대 여행자
- 혼자 또는 소수 인원으로 여행하며, 현장에서 이야기를 발견하고 싶은 사람
- 여행 후 기록을 남기고 싶지만 블로그 작성이 번거로운 사람

### 제약 조건

- **지역 한정:** 제주도 내 설화 데이터가 매핑된 장소에서만 동작한다. 설화 데이터가 없는 장소(공항, 대형마트 등)는 동행자가 말을 걸지 않는다.
- **네트워크 의존:** AI 동행자 채팅은 GPT-4o API를 사용하므로 인터넷 연결이 필요하다. 오프라인 환경에서는 채팅 불가.
- **iOS 전용:** 현재 iOS 17.0 이상 기기에서만 동작한다.
- **배터리:** 백그라운드 GPS 추적은 배터리 소모를 유발한다. 사용자에게 사전 안내 필요.

## 6. Value Proposition

### 사용자가 해결하려는 문제 (Jobs to be done)

1. **탐험 욕구:** "이 장소에 어떤 이야기가 있는지 알고 싶다." — 지금은 직접 검색해야 알 수 있다.
2. **기록 욕구:** "이 여행을 나중에 다시 느끼고 싶다." — 지금은 사진 외에 맥락 있는 기록이 없다.
3. **차별화 욕구:** "다른 사람들과 다른, 나만의 제주 여행을 하고 싶다." — 지금은 모두 비슷한 관광 동선을 따른다.

### 우리가 제공하는 것

| 기존 여행 앱 | 우리 앱 |
|---|---|
| 장소 정보 (별점, 운영시간, 사진) | 장소에 깃든 설화와 이야기 |
| 사용자가 직접 체크인 | 도착하면 앱이 먼저 말을 걸어옴 |
| 모든 사용자에게 같은 정보 | 설화 취향에 맞는 캐릭터가 다르게 이야기함 |
| 사진 위주 기록 | 대화 기반 개인화 여행 일지 자동 생성 |

### 경쟁 우위

- **설화 데이터:** 제주 설화 전체를 GPS 좌표와 매핑한 데이터베이스는 우리 앱만 보유.
- **캐릭터 개인화:** 설화 취향에 따라 동행자 캐릭터가 달라지는 구조는 타 앱에 없음.
- **대화 기반 일지:** 여행 중 나눈 대화가 그대로 기록이 되는 방식은 기존 여행 앱과 완전히 다른 접근.

## 7. Solution

### 7.1 UX Flow

#### 여행 중 (5단계)

```
[1단계] 여행 시작
- 진입점: 코스 확정 화면 하단 "오늘 탐험 시작" 버튼
- 탐험 모드 진입 시 오늘(Day N) 방문 장소들이 지도에 핀으로 표시
- 미방문 핀: 주황색 / 방문완료 핀: 회색 (시각 구분)
- 백그라운드 위치 추적 시작 (사용자에게 알림)

[2단계] 이동 중
- 지도 중심 전체화면
- 이동 경로가 점선으로 실시간 표시 (폴라스텝 방식)
- 하단 플로팅 카드: 다음 목적지 이름 + 직선거리 + "카카오맵으로 길찾기" 딥링크

[3단계] 도착 감지
- 조건: GPS 지오펜스 100m 반경 진입 AND 30초 이상 체류 (오탐 방지)
- 감지 시: 진동 + 효과음
- 전체화면 도착 오버레이: "○○에 도착했습니다 — 이야기가 기다리고 있어요"
- 탭 1회 → AI 동행자 채팅 화면으로 전환

[4단계] AI 동행자 채팅
- 동행자가 먼저 말을 걺 (해당 장소의 설화를 시스템 컨텍스트로 로드)
- 사용자와 자유 대화 (멀티턴)
- 채팅 내용은 로컬 SQLite에 실시간 저장
- 탐험 화면 우하단 채팅 버튼으로 언제든 재접근 가능

[5단계] 하루 종료
- "오늘 탐험 마치기" 버튼 탭
- 오늘 방문 장소 요약 카드 + 나눈 대화 수 표시
- 채팅 기록 저장 확정 → 다음 Day 안내
```

#### 여행 후 (1단계)

```
[6단계] 여행 일지 생성
- 모든 Day 탐험 완료 후 "여행 마치기" 버튼
- 방문 장소 목록 + 채팅 기록 전체를 백엔드 API에 전송
- GPT-4o가 대화 맥락을 분석해 개인화된 여행 일지 생성
- 일지는 텍스트 형태로 화면에 표시 + 로컬 저장
```

### 7.2 핵심 기능 상세

#### 기능 1: AI 설화 동행자

코스 설계 단계에서 선택한 설화 취향(category_scores)의 최고 점수 카테고리를 기준으로 동행자 캐릭터가 자동 배정된다.

| 최고 점수 카테고리 | 동행자 캐릭터 | 말투 특징 |
|---|---|---|
| 무속신화·신격 전승 | 당신(堂神) / 심방 | 신성하고 묵직한 어투, 신화적 언어 |
| 생활민담·교훈담 | 도깨비 | 장난기 있고 익살스러운 어투 |
| 마을 공동체 전승 | 마을 할망 | 따뜻하고 친근한 제주 사투리 |
| 해양·어촌 전승 | 영등신 / 해녀 선배 | 바다 냄새 나는 거칠고 시원한 어투 |
| 초자연 존재담 | 도체비 | 불안정하고 기묘한 어투, 반전 있는 말 |

**시스템 프롬프트 구조:**
```
[캐릭터 페르소나 정의]
[현재 장소 이름 + 설화 데이터 (folklore_pins 최대 3개)]
[사용자 설화 취향 (category_scores)]
[대화 가이드라인: 설화를 강요하지 않고 자연스럽게 풀어낼 것]
```

#### 기능 2: GPS 지오펜스 도착 감지

- **기술:** iOS CoreLocation `CLCircularRegion`
- **반경:** 100m (Detour 방식 참고, 제주 설화 장소 특성에 맞게 조정)
- **오탐 방지:** 반경 진입 후 30초 이상 체류 시에만 트리거
- **배터리:** `CLLocationManager`의 `allowsBackgroundLocationUpdates = true` + `pausesLocationUpdatesAutomatically = false`
- **미등록 장소:** 설화 데이터가 없는 장소(공항, 항구 등)는 감지해도 동행자가 말을 걸지 않음

#### 기능 3: 채팅 기록 저장

- **저장 위치:** MVP에서는 로컬 SQLite (앱 삭제 시 소실 — 허용 가능한 MVP 제약)
- **저장 단위:** 장소별 대화 세션 (place_id + day + timestamp + 메시지 배열)
- **추후:** 서버 저장 및 동기화

#### 기능 4: 여행 일지 자동 생성

- **입력:** 방문 장소 목록 + 각 장소의 채팅 기록 전체
- **처리:** 백엔드 `/travel/journal` API → GPT-4o 요약
- **출력:** 2~4문단 분량의 한국어 여행 일지 텍스트
- **프롬프트 방향:** "이 사람만의 제주 탐험기"를 쓰는 스토리텔러 관점

### 7.3 기술 스택

| 영역 | 기술 |
|---|---|
| iOS 위치 감지 | CoreLocation CLCircularRegion |
| iOS 채팅 UI | SwiftUI (커스텀 채팅 뷰) |
| iOS 로컬 저장 | SQLite (기존 db.py 서비스 활용) |
| 백엔드 채팅 API | FastAPI + LangChain + GPT-4o |
| 백엔드 일지 API | FastAPI + GPT-4o structured output |
| 설화 데이터 | folklore_gps.json (기존 데이터 재활용) |

### 7.4 가정 (Assumptions)

아래는 아직 검증되지 않은 가정들이다. 구현 전 또는 중에 확인이 필요하다.

| # | 가정 | 검증 방법 |
|---|---|---|
| A1 | 제주 도심(북부)에서도 GPS 오차가 100m 이내 | 현장 테스트 |
| A2 | 30초 체류 조건이 오탐을 충분히 줄여준다 | 현장 테스트 (차로 지나치는 경우 포함) |
| A3 | 사용자가 동행자 채팅을 3턴 이상 지속한다 | 프로토타입 테스트 |
| A4 | 채팅 기록으로 의미 있는 일지가 생성된다 | GPT-4o 프롬프트 실험 |
| A5 | 배터리 소모가 여행 하루치(8시간)를 버틴다 | 기기 테스트 |

## 8. Release

### MVP (다음 미팅까지 목표)

**포함:**
- 여행 시작 화면 (탐험 모드 진입 + Day 장소 지도 표시)
- 이동 중 화면 (지도 + 다음 목적지 카드)
- GPS 도착 감지 (CLCircularRegion, 100m + 30초)
- 도착 오버레이 + AI 동행자 채팅 화면
- 채팅 로컬 저장 (SQLite)
- 하루 종료 요약 화면
- 여행 일지 생성 (백엔드 API + 화면 표시)

**제외 (추후):**
- 일지 SNS 공유 카드 생성
- 오프라인 캐시 (네트워크 없이 설화 텍스트 제공)
- 채팅 기록 서버 동기화
- 다른 결말 재방문 (같은 코스, 다른 대화 루트)
- 설화 조각 수집 게임화 (Codex)

### 개발 순서 제안

1. **1주차:** CoreLocation 지오펜스 감지 구현 + 도착 오버레이 UI
2. **1주차:** AI 동행자 백엔드 API (`/travel/chat`) — 캐릭터 페르소나 + 설화 컨텍스트 주입
3. **2주차:** 채팅 UI + 로컬 저장
