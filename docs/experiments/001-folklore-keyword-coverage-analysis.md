## 기존 키워드의 문제

초기 키워드는 직관으로 설계:

```python
THEME_KEYWORDS = {
    "신화":      ["신화", "본풀이", "설문대할망", "천지왕", "창세", "신"],
    "도깨비":    ["도깨비", "도체비", "요괴", "귀신", "괴물", "혼"],
    "사랑과이별": ["사랑", "이별", "부부", "연인", "그리움", "눈물"],
    "바다해녀":  ["해녀", "바다", "어부", "용왕", "물속", "해신"],
    "오름자연":  ["오름", "한라산", "폭포", "숲", "자연", "동물"],
}
```

실제 텍스트 스캔 결과 발견된 문제:
- `오름자연`: "오름" 직접 언급 거의 없음. 자연은 배경으로만 등장
- `바다해녀`: 순수 바다/해녀 중심 파일 30~40개 수준. 단어 "해녀" 출현 파일 자체가 적음
- `도깨비`: 제주어로 "도체비"가 표준 표기. "도깨비"만으로 검색 시 누락 발생
- 쿼리 형태가 단어 나열이라 OpenAI 임베딩의 문맥 파악 능력을 활용 못함

## 개선 방안 적용

단어 나열 → **문장형 쿼리**로 변경 (OpenAI 임베딩 문맥 활용):

```python
THEME_QUERIES = {
    "바다해녀": (
        "제주 바다 해녀 어촌 생활 이야기. "
        "잠수 물질 전복 소라 멸치 멜 어장 고기잡이 바당. "
        "용왕 영등할망 어부 선박 해신 풍어제. "
        "바다에서 살아가는 제주 사람들 삶."
    ),
    ...
}
```

근거: 임베딩 모델은 단어 나열보다 문맥이 있는 문장에서 의미 벡터를 더 정확하게 생성.  
특히 `text-embedding-3-small`은 긴 문맥 입력에서 성능 향상이 보고됨.
