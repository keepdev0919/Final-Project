## 1. 왜 따로 문서화하는가

현재 코스 추천에 바로 사용할 수 있는 설화는 GPS가 붙은 228개입니다.  
하지만 나머지 277개 설화·민담도 의미가 없는 것은 아닙니다.

이 데이터는 앞으로 다음 용도로 다시 쓸 수 있습니다.

- 챗봇·RAG 답변 보강
- 장소 상세 페이지의 보조 이야기
- 코스 설명문 보강
- GPS 복원 후보 관리

따라서 지금 단계에서는 복잡한 분류보다, **핵심 3항목만 남기는 최소 문서화**가 더 실용적입니다.

---

## 2. 설화 1개당 남기는 3개 항목

각 설화·민담은 아래 3개만 기록합니다.

1. `core_category`
   - 이 설화가 어떤 유형의 이야기인가

2. `spatial_clue`
   - 텍스트 안에서 읽히는 가장 강한 공간·배경 단서가 무엇인가

3. `service_use`
   - 현재 서비스에서 이 설화를 어디에 우선 활용할 것인가

즉 각 항목은 다음 질문에 대응합니다.

```text
이 설화는 어떤 이야기인가?
어디와 연결될 가능성이 있는가?
서비스에서 어디에 쓸 것인가?
```

## 4. 활용 원칙

현재 단계에서는 비GPS 설화를 코스 생성에 직접 투입하지 않습니다.

대신 다음처럼 씁니다.

- `챗봇·RAG 보강`
  - 위치를 확정할 수 없지만, 카테고리/분위기/존재 설명에는 도움이 되는 설화

- `장소 상세/코스 설명 보강`
  - 직접 코스에 넣진 않지만, 특정 장소나 지역 설명에 함께 붙이면 좋은 설화

- `GPS 복원 후보`
  - 공간 단서가 비교적 강해서 나중에 수동 검토로 위치를 복원할 가능성이 있는 설화

즉 비GPS 설화는 **경로 데이터**가 아니라 **콘텐츠 자산**으로 우선 관리합니다.

## 5. 현재 단계의 결론

1. GPS 없는 설화·민담도 버릴 데이터는 아닙니다.
2. 다만 지금은 3항목만 남겨 최소 문서화하는 것이 가장 효율적입니다.
3. 현재 기준으로는 `챗봇·RAG 보강` 역할이 가장 크고, 일부는 `GPS 복원 후보`로 별도 관리할 가치가 있습니다.
4. 따라서 이 데이터는 코스 추천 핵심 풀과 분리된 **보조 콘텐츠 레이어**로 보는 것이 적절합니다.
