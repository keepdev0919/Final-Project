# 003. 테마 키워드 직관 설계 → 데이터 기반 재설계

**날짜**: 2026-04-08  
**관련 실험**: experiments/001-folklore-keyword-coverage-analysis.md

---

## 배경

ChromaDB 벡터 검색 쿼리에 사용하는 `THEME_KEYWORDS`가 직관으로 설계되어,  
실제 설화 텍스트와 괴리가 있을 수 있다는 문제 제기.

---

## 문제 정의

사용자 선택 → 내부 테마 → **ChromaDB 쿼리** → 설화 검색 → 장소 추출 → 코스 생성

이 파이프라인에서 "ChromaDB 쿼리 품질 = 코스 추천 품질"이므로,  
키워드가 실제 데이터를 얼마나 커버하느냐가 서비스 품질을 직접 결정.

---

## 분석 방법

legend 182개 + folktale 323개 = 505개 파일 전수 텍스트 스캔.  
빈출 단어 추출 및 5개 테마별 분류.

---

## 결정

1. **키워드를 실제 텍스트 출현 단어로 교체**  
   예) "오름자연" 테마에 "오름"(출현 적음) 대신 "설문대할망", "지명유래", "고종달" 추가

2. **쿼리 형태를 단어 나열 → 문장형으로 변경**  
   단어: `"오름 한라산 폭포 숲 자연 동물 바람"`  
   문장: `"제주 오름 자연 지형 지명 유래 이야기. 한라산 설문대할망 백록담..."`

3. **제주어 표기 반영**  
   도깨비 → 도체비 / 바다 → 바당 / 슬프다 → 설운·서로운

---

## 결과

예상 커버리지: **82.6%** (417/505개)  
나머지 17.4%는 일상·해학·교훈 민담으로 5개 테마 구조에서 귀속이 구조적으로 어려운 파일.

---
