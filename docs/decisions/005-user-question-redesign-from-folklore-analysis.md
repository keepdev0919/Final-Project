## 왜 질문을 다시 설계하는가

기존 질문 구조는 다음과 같았습니다.

1. 분위기
2. 장소
3. 일수
4. 이동수단

이 구조는 UX 관점에서는 직관적이었지만, 실제 설화 데이터에서 도출된 카테고리를 정확히 가려내기에는 한계가 있었습니다.

특히:
- 일부 답변은 결과가 지나치게 단조롭게 매핑되었습니다.
- 질문이 먼저 있고 카테고리가 뒤따라오는 구조였습니다.
- 설화의 실제 반복 축인 `무속신화`, `생활민담`, `마을 공동체`, `지명·지형`, `해양`, `초자연`, `가족·인간사`를 충분히 반영하지 못했습니다.

따라서 이제는 질문을 먼저 예쁘게 만드는 것이 아니라,  
**설화 분석으로 도출한 내부 카테고리를 가장 잘 구분할 수 있도록 질문을 역설계**해야 합니다.

## 내부 카테고리 v1

현재 GPS 사용 가능 설화 228개 1차 태깅 결과를 기준으로 내부 카테고리는 다음 7개를 사용합니다.

1. `무속신화·신격 전승`
2. `생활민담·교훈담`
3. `마을 공동체 전승`
4. `지명·지형 유래`
5. `해양·어촌 전승`
6. `초자연 존재담`
7. `가족·인간사 서사`

이 중 질문 설계에서는 크게 두 축을 분리해서 다룹니다.

- **서사 축**
  - 무속신화·신격 전승
  - 초자연 존재담
  - 가족·인간사 서사
  - 생활민담·교훈담

- **공간/전승 축**
  - 마을 공동체 전승
  - 지명·지형 유래
  - 해양·어촌 전승

## 질문 세트 v1

### Q1. 어떤 이야기 결이 끌리나요?

사용자용 선택지:
- 신성한 신과 당의 이야기
- 으스스하고 신비한 존재 이야기
- 사람 사이의 관계와 감정이 남는 이야기
- 재치 있고 교훈적인 민담

내부 점수 매핑:

| 답변 | 내부 카테고리 점수 |
|------|------------------|
| 신성한 신과 당의 이야기 | 무속신화·신격 전승 +3, 마을 공동체 전승 +1 |
| 으스스하고 신비한 존재 이야기 | 초자연 존재담 +3, 해양·어촌 전승 +1 |
| 사람 사이의 관계와 감정이 남는 이야기 | 가족·인간사 서사 +3, 생활민담·교훈담 +1 |
| 재치 있고 교훈적인 민담 | 생활민담·교훈담 +3, 가족·인간사 서사 +1 |

### Q2. 어떤 배경이 더 끌리나요?

사용자용 선택지:
- 바다와 해안 마을
- 마을과 당, 사람들이 살아온 동네
- 산, 오름, 지형이 살아 있는 곳
- 크게 상관없어요

내부 점수 매핑:

| 답변 | 내부 카테고리 점수 |
|------|------------------|
| 바다와 해안 마을 | 해양·어촌 전승 +3, 마을 공동체 전승 +1 |
| 마을과 당, 사람들이 살아온 동네 | 마을 공동체 전승 +3, 무속신화·신격 전승 +1, 가족·인간사 서사 +1 |
| 산, 오름, 지형이 살아 있는 곳 | 지명·지형 유래 +3, 무속신화·신격 전승 +1 |
| 크게 상관없어요 | 가산점 없음 |

### Q3. 어떤 방식으로 이어지는 이야기가 좋나요?

사용자용 선택지:
- 장소의 유래나 기원을 알게 되는 이야기
- 마을 사람들이 함께 믿고 전해온 이야기
- 한 사람이나 가족에게 일어난 이야기
- 인간 말고 다른 존재가 중심인 이야기

내부 점수 매핑:

| 답변 | 내부 카테고리 점수 |
|------|------------------|
| 장소의 유래나 기원을 알게 되는 이야기 | 지명·지형 유래 +3, 무속신화·신격 전승 +1 |
| 마을 사람들이 함께 믿고 전해온 이야기 | 마을 공동체 전승 +3, 무속신화·신격 전승 +1 |
| 한 사람이나 가족에게 일어난 이야기 | 가족·인간사 서사 +3, 생활민담·교훈담 +1 |
| 인간 말고 다른 존재가 중심인 이야기 | 초자연 존재담 +2, 무속신화·신격 전승 +1, 해양·어촌 전승 +1 |

### Q4. 여행 기간은 며칠인가요?

- 대중교통·도보

## 점수 계산 방식

각 답변에 따라 내부 카테고리 점수를 누적합니다.

예시:

```text
Q1 = 신성한 신과 당의 이야기
Q2 = 산, 오름, 지형이 살아 있는 곳
Q3 = 장소의 유래나 기원을 알게 되는 이야기

결과:
- 무속신화·신격 전승 = 3 + 1 + 1 = 5
- 지명·지형 유래 = 3 + 3 = 6
- 마을 공동체 전승 = 1
```

이 경우 최종 후보 카테고리는:
- 1위: `지명·지형 유래`
- 2위: `무속신화·신격 전승`

으로 정하고, 이후 설화 후보 선택 단계에서 이 두 카테고리를 함께 반영합니다.

## 최종 사용 원칙

질문 결과는 단일 카테고리 하나를 강제로 고르는 데 쓰지 않습니다.  
대신 다음과 같이 사용합니다.

1. 질문 답변으로 7개 내부 카테고리 점수를 계산합니다.
2. 상위 2개 카테고리를 우선 후보군으로 삼습니다.
3. 해당 카테고리에 속한 GPS 설화들 중에서 지역, 거리, 일수, 이동수단 조건을 반영해 후보를 줄입니다.
4. 이후 에이전트가 비짓제주 데이터와 결합해 실제 코스를 설계합니다.

즉 질문은 "테마 하나를 뽑는 절차"가 아니라,  
**설화 후보군을 점수 기반으로 좁히는 단계**가 됩니다.

- 특히 `전승 방식` 질문이 추가되면서, 단순한 분위기 선택보다 데이터와 더 잘 연결됩니다.
