# 001. 지도 중심 UI → Taste Discovery 온보딩 UX 재설계

**날짜**: 2026-04-08  
**관련 커밋**: ed85aa3 이후 변경사항

---

## 배경

초기 앱 구조는 메인 화면이 지도(HomeView)였음.  
지도에 설화 핀이 클러스터링되어 표시되고, 핀을 누르면 설화 전문을 볼 수 있는 구조.

**문제점**:
- 사용자가 처음 앱을 열었을 때 지도만 덩그러니 보임 → "이게 뭐하는 앱인지" 불명확
- 서비스 본질(설화 기반 여행 코스 추천)이 전달되지 않음
- 설화 카테고리(신화/도깨비/사랑과이별...)를 직접 고르는 방식 → 사용자가 카테고리 체계를 모름

---

## 분석

서비스 본질, MVP, 타겟 사용자를 재정의:

| 항목 | 결정 |
|------|------|
| 핵심 기능 | 설화 기반 여행 코스 추천 |
| 타겟 사용자 | 제주 여행 계획 중인 사람 |
| 진입점 | 사용자가 자신의 취향을 모르는 상태에서 시작 |

사용자가 "나는 신화 테마가 좋아"를 모르는 상태이므로,  
자연어 수준의 질문(분위기, 장소 선호)으로 내부 테마를 추론해야 한다는 결론.

---

## 결정: Taste Discovery 패턴 도입

4단계 온보딩 퀴즈를 통해 내부 테마 매핑:

```
Q1. 어떤 분위기가 끌려요?
    🌙 신비롭고 으스스한 / 🌸 따뜻하고 감동적인 / ⛩️ 웅장하고 신성한 / 🎣 사람들의 삶 이야기

Q2. 주로 어디가 좋아요?
    🌊 바다 / 🌿 오름·산 / 🏘️ 마을 / ✨ 상관없어요

Q3. 며칠이에요?  (1일 / 2일 / 3일 / 4일+)

Q4. 이동수단은요?  🚗 렌트카 / 🚶 대중교통·도보
```

**사용자 언어 → 내부 테마 매핑 (mapTheme)**:

| 분위기 | 장소 | 내부 테마 |
|--------|------|----------|
| 신비롭고 으스스한 | 바다 | 바다해녀 |
| 신비롭고 으스스한 | 그 외 | 도깨비 |
| 웅장하고 신성한 | (무관) | 신화 |
| 따뜻하고 감동적인 | 바다 | 바다해녀 |
| 따뜻하고 감동적인 | 오름·산 | 오름자연 |
| 따뜻하고 감동적인 | 마을/기타 | 사랑과이별 |
| 사람들의 삶 이야기 | 바다 | 바다해녀 |
| 사람들의 삶 이야기 | 오름·산 | 오름자연 |
| 사람들의 삶 이야기 | 마을 | 사랑과이별 |
| 사람들의 삶 이야기 | 상관없어요 | 바다해녀 |

## 탭 구조 변경

| 이전 | 이후 |
|------|------|
| 지도 / 코스 추천 / 내 코스 / 챗봇 | 코스 만들기(TasteDiscovery) / 내 코스 / 챗봇 |

지도 탭 제거 → 코스 결과 화면 내에서 지도를 보여주는 방식으로 전환.
