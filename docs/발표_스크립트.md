### 🟦 슬라이드 06 — 코스 추천 알고리즘 (1분)

> "사용자는 **세 가지를 입력**합니다. 제주 어느 지역, 설화 카테고리 1·2순위, 여행 일자.
>
> 백엔드는 먼저 **지역과 일수 조건**으로 매핑 보유 코스 풀에서 **후보 50개**를 추립니다. 이 단계에선 카테고리 점수는 안 봅니다.
>
> 그 다음 **50개 각각에 점수를 매깁니다**. 각 코스의 매핑 카테고리 비율을 계산하고, 사용자 점수와 곱한 합산이 코스 점수입니다.
>
> **점수가 양수인 풀에서 무작위 3개**를 뽑아 보여줍니다. 점수가 0인 코스는 사용자 취향과 안 맞는 거라 자동 탈락합니다.
>
> 이 전 과정에 **LLM은 쓰지 않습니다**. SQL과 결정론적 계산만으로 응답 속도와 비용을 잡았습니다."

**핵심 키워드**: 3 입력 / 50개 후보 / 점수 / 무작위 3개 / LLM 0회

### 🟦 슬라이드 07 — 코스 점수 계산 예시 (1분)

> "실제 코스 한 건으로 점수 계산을 보여드리겠습니다.
>
> 「조금 이른 여름휴가」 코스에는 애월해안도로와 사려니숲 두 장소가 있고, 매핑된 설화·민담이 **총 8건**입니다.
>
> 카테고리별 비율을 보면 **생활민담·교훈담이 37.5%**, 무속신화 25%, 나머지 12.5%씩입니다.
>
> 사용자가 **생활민담을 1순위 3점, 무속신화를 2순위 1점**으로 입력했다면, 곱셈하면 3×37.5%=1.125, 1×25%=0.25, 합계 1.375에 10을 곱해 **코스 점수는 13.75점**입니다.
>
> 양수이니 풀에 들어가서 무작위 3개에 뽑힐 수 있게 됩니다."

**핵심 키워드**: 8건 매핑 / 37.5% / 13.75점

## 🎯 Q&A 대비 — 자주 나올 5가지 질문

### Q1. "AI 에이전트는 어디 쓰였나요?"
> "도구 자율 호출의 ReAct 에이전트는 시스템에서 제거했고, 현재는 **RAG·페르소나 프롬프트·구조화 출력** 조합으로 구현했습니다. 핵심은 LLM 호출의 예측 가능성과 응답 속도이고, ReAct는 응답 지연이 누적되어 사용자 이탈 위험이 있다고 판단했습니다."

### Q2. "왜 GPT를 코스 추천에 안 썼나요?"
> "LLM은 **비용·지연·환각·재현 가능성** 네 가지 트레이드오프를 동반합니다. 코스 추천은 사전 정제된 매핑과 결정론적 점수만으로 해결 가능한 영역이라 LLM을 의도적으로 배제했습니다. LLM은 동행자 채팅·일지 생성처럼 자연어 창의성이 핵심인 곳에만 활용했습니다."

### Q3. "매핑 테이블에서 GPS 보조 매핑을 왜 제외했나요?"
> "GPS 8km 보조 매핑은 '근처에 있다 ≠ 그 명소의 설화'라는 정합성 우려가 있어 코스 추천에서 제외했습니다. **사전 매핑 테이블에 source 컬럼**을 두고 `source != 'gps_assist'` 필터를 적용해, 데이터는 보존하면서 사용자 노출에서만 제외했습니다."

### Q4. "ChromaDB는 정확히 무엇이고 1,262 청크의 의미는?"
> "ChromaDB는 텍스트를 벡터로 저장해 의미가 비슷한 것을 찾는 RAG 검색 전용 DB입니다. 505건 전체를 적재하지만, **장소 매핑된 397 설화·민담의 1,262 청크만 동행자 채팅 검색 대상**으로 제한했습니다. `is_mapped` 메타데이터 필터로 진짜 정합성을 보장합니다."

### Q5. "왜 페르소나가 5종인가요?"
> "사용자 입력 카테고리가 5종이라 1:1 매칭으로 자연스럽게 연결됩니다. 단일 챗봇은 톤만 표면적으로 바뀌지만, **5종은 시스템 프롬프트와 어휘·퓨샷이 모두 독립**이라 같은 RAG 결과를 다섯 가지 사고 흐름으로 풀어냅니다. 콘텐츠 다양성을 페르소나 분리로 만들었습니다."
